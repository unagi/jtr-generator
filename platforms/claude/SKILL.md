# 日本の履歴書生成Skill

## Overview

JIS規格準拠の日本の履歴書をPDF形式で生成します。
対話的な情報収集またはYAML/JSONファイルからの生成が可能です。

## Capabilities

- 入力形式: YAML, JSON
- 出力形式: PDF（A4サイズ、和暦/西暦切り替え可能）
- カスタムフォント対応（デフォルト: BIZ UDMincho）
- JSON Schema準拠のバリデーション
- エラーメッセージは日本語で明確に説明

## Required Data Schema

### 必須フィールド
- **personal_info**: 氏名、氏名かな、生年月日、性別、住所、電話番号、メールアドレス
- **education**: 学歴（配列、最低1件）
- **work_history**: 職歴（配列、0件以上）

### オプショナルフィールド
- **contact_info**: 連絡先（現住所以外に連絡を希望する場合）
- **qualifications**: 資格・免許（配列）
- **additional_info**: 志望動機、自己PR、本人希望欄

### 日付形式
- 生年月日: YYYY-MM-DD（例: 1990-04-01）
- 年月: YYYY-MM（例: 2010-04）
- ISO 8601形式を厳守

### 正式名称の使用
- 学校名: 「東京都立○○高等学校」（「高校」と略さない）
- 会社名: 「株式会社○○」または「○○株式会社」（正式名称、「㈱」禁止）
- 資格名: 「基本情報技術者試験 合格」（正式名称）

## Data Collection Rules

あなたは日本の履歴書データを対話的に収集し、PDF形式で生成するアシスタントです。

1. **必須フィールドの確認**: 不足している場合は必ず質問してください
2. **日付形式の統一**: すべての日付はISO 8601形式で収集。ユーザーが和暦で回答した場合は西暦に変換してください
3. **正式名称の使用**: 略語禁止（上記参照）
4. **サンプルデータの参照**: ユーザーが形式を尋ねた場合、examples/sample_resume.yamlを参照

## Examples

1. 対話的生成:
   「履歴書を作成してください。氏名は山田太郎です。生年月日は1990年4月1日です。」
   → 不足している情報を対話的に収集します

2. YAMLファイルからの生成:
   [YAMLファイルを添付] 「このファイルから履歴書を生成してください。」
   → ファイルを読み込み、バリデーション後にPDF生成

3. オプション指定:
   「和暦表記で履歴書を作成してください。」
   → config.yamlのデフォルト設定を上書き

4. サンプルデータの参照:
   「サンプルデータの形式を教えてください。」
   → examples/sample_resume.yamlを参照

## Error Handling

- **フォントファイル不在時**: 「フォントファイルが見つかりません。config.yamlのfonts設定を確認するか、デフォルトフォント（BIZ UDMincho）のパスを確認してください。」
- **JSON Schemaバリデーション失敗時**: 「データ検証に失敗しました。[具体的なエラー内容] を修正してください。」
- **日付形式エラー**: 「日付形式が不正です。YYYY-MM-DD形式（例: 1990-04-01）で指定してください。」

## Execution Flow

1. ユーザーから情報を収集
2. 収集したデータをYAML/JSON形式に変換
3. main.py内のmain()関数を呼び出してPDF生成
4. 生成されたPDFファイルのパスをユーザーに返す

## Notes

- 個人情報の取り扱いに注意してください
- 生成されたPDFはセッション終了後に自動削除されます
- エラーメッセージは必ず日本語で表示してください
